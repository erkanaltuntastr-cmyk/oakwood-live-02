# Oakwood Learning Hub — v0 Design Input Instructions

## Purpose
This folder contains the visual source of truth for creating a web-first UI prototype for Oakwood Learning Hub.

## Product
Oakwood Learning Hub is a calm, premium, family-focused UK education web app for parents and students.

Tagline:
Supporting families, empowering students, advancing education.

## v0 Task
Create UI/UX prototypes only.
Do not create backend logic.
Do not create database logic.
Do not create authentication infrastructure.
Do not create API endpoints.
Do not redesign the product architecture.

## Brand Rules
Use the supplied Oakwood assets directly.
Do not redesign the Oakwood logo.
Do not invent a new brand style.
Do not replace supplied PNG brand assets with generic icons.
Do not use react.svg or vite.svg.
Do not use bright childish colours.
Do not use robot / sci-fi AI visuals.
Do not use shields, badges, crests, or school-army style emblems.

## Visual Direction
- Web-first, not mobile-first
- Spacious landing page
- Sidebar + topbar dashboard shell
- Large profile/avatar cards
- Warm off-white backgrounds
- Soft green / sage / beige palette
- Organic botanical Oakwood feeling
- Rounded cards
- Light borders
- Gentle shadows
- Clear hierarchy
- Calm premium dashboard style

## Required First Prototype Screens
1. Welcome / landing page
2. Login page
3. Profile select page
4. Parent dashboard
5. Student dashboard

## Asset Usage
Use BRAND_ASSET_USAGE.md as the usage map.
Use Oakwood Learning Hub — Marka, Stil ve Uygulama Stratejisi.MD as the style guide.

## Important
This is a visual contract for later implementation in the existing Vite/React app.
Do not create a new backend.
Do not create a new full-stack app.
Do not change the brand.
Do not simplify the UI into generic SaaS style.
